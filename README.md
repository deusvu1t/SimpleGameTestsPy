# SimpleGameTestsPy

`SimpleGameTestsPy` — небольшой учебный Python-проект с простой игровой логикой и набором автотестов на `pytest` + `allure`.

## Описание проекта

В центре проекта находится класс `Player`, который хранит количество монет игрока и взаимодействует с двумя вспомогательными компонентами:

- `GameAnalytics` — сохраняет игровые события;
- `Logger` — записывает действия игрока в память.

Это минимальный пример, на котором удобно показывать:

- как разделять игровую логику по модулям;
- как писать unit-тесты для бизнес-логики;
- как оформлять шаги и сценарии в Allure-отчётах.

## Структура проекта

```text
SimpleGameTestsPy/
├── core/
│   ├── analytics.py
│   └── player.py
├── tests/
│   └── test_player.py
├── utils/
│   └── logger.py
├── pytest.ini
└── requirements.txt
```

### Основные файлы

- `core/player.py` — содержит класс `Player` с методами `add_coins()` и `spend_coins()`.
- `core/analytics.py` — содержит класс `GameAnalytics`, который накапливает события в списке `events`.
- `utils/logger.py` — содержит класс `Logger`, который сохраняет сообщения в списке `logs`.
- `tests/test_player.py` — тестирует игровую логику и использует `allure.feature`, `allure.story` и `allure.step`.
- `pytest.ini` — добавляет корень проекта в `pythonpath`, чтобы тесты запускались без дополнительной настройки.

## Как работает логика игрока

### Добавление монет

Метод `add_coins(amount)`:

1. увеличивает баланс игрока;
2. отправляет событие `coins_added` в аналитику;
3. добавляет сообщение `Coins added: <amount>` в лог.

### Трата монет

Метод `spend_coins(amount)`:

- если монет достаточно:
  - уменьшает баланс;
  - пишет событие `purchase_success`;
  - пишет лог `Purchase success: <amount>`;
  - возвращает `True`;
- если монет недостаточно:
  - пишет событие `purchase_failed`;
  - пишет лог `Purchase failed: <amount>`;
  - возвращает `False`.

## Что покрывают тесты

Сейчас в проекте есть 4 теста:

- проверка, что после `add_coins(100)` баланс игрока становится равен `100`;
- проверка успешной траты монет после пополнения;
- проверка, что успешная покупка записывается в аналитику;
- проверка, что после добавления монет появляется ожидаемая запись в логах.

## Установка

Рекомендуемая версия Python: **3.10+**.

Установить зависимости можно так:

```bash
pip install -r requirements.txt
```

## Запуск тестов

Базовый запуск:

```bash
pytest
```

Более подробный вывод:

```bash
pytest -v
```

## Генерация Allure-результатов

Чтобы сохранить результаты для Allure-отчёта:

```bash
pytest --alluredir=allure-results
```

Если у вас установлен Allure CLI, отчёт можно открыть командой:

```bash
allure serve allure-results
```
